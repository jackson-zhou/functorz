一、完整平台全景架构 Full Platform Architecture

低代码平台的本质，是「用一份声明式描述把 App 讲清楚（界面 / 数据 / 逻辑），再由引擎去解释并运行它」。完整平台由五层、三大引擎构成，横跨设计态、运行态、数据、集成与平台底座——这也是原 PC 版多年建设所沉淀的复杂度。

二、分期落地路线 Phased Roadmap

完整平台按「可交付、可验收、可独立计价」的节奏分期推进。每一期都是上一期的自然生长，甲方每期都能拿到可用成果，风险与投入逐期释放。

第一期 · 本报价
三引擎骨架 + 拖拉搭建 + 电商样板
第二期
数据模型设计器 + 增删改查 + 事务
第三期
微信 登录 / 支付 / 分享 + 连接器 + 密钥
第四期
算子市场 + 复杂流程 + 小程序双端对齐
第五期 +
权限多租户 + 模板市场 + 调试器 + 协作

三、第一期实现范围 Phase 1 Scope

第一期是全景图主干的一条最薄纵切：从「拖拉搭界面 / 拖拉搭流程」，到「三大引擎运行」，再到「后端数据」，用一个电商场景（登录 + 商品列表）端到端跑通。范围可控，但把整套架构的骨架真正立了起来。

五、三个月排期 Timeline

  1. 第 1 月：共享框架（Schema / 表达式 / 算子接口 / 状态存储）+ 界面布局引擎 + 拖拉页面搭建器。

本建议书仅覆盖第一期。完整平台为多期框架合同，各期独立范围、独立验收、独立计价。
技术栈：Taro（H5 + 微信小程序）· React · RisingWave · Snurr / CompileFlow（图化执行）。
